#  Personal Health Predictive Analytics App -- HealthJourney AI

An End-to-End (Full-Stack) Predictive Health Analytics system that leverages a Multi-Output Machine Learning model to predict a user's **Quality of Sleep** and **Stress Level** based on daily biometric data. Built with a responsive **Flutter** mobile client and a high-performance **FastAPI** backend engine.

---

### 🔄 My Workflow
My development cycle followed a strict software engineering pipeline:
1. **Data Exploration & Strategy:** Cleaned and evaluated the *Sleep Health and Lifestyle Dataset* using Pandas, analyzed correlation matrices, and handled sparse categorical encoding.
2. **Model Selection & Training:** Evaluated multiple architectures (Linear Regression vs. Random Forest vs. XGBoost) before choosing the ideal regressor, which finally was XGBOOST with a r2 score of 0.90 and mse of 0.18 using training dataset.
3. **Backend:** Wrapped the saved mathematical pipeline file (`.pkl`) inside a FastAPI gateway to handle data validation, where I used A helpI to handle API codes.
4. **Frontend:** Developed a clean UI in Flutter to capture real-time biometrics and manage asynchronous network states seamlessly with the help of AI.

### ⚠️ Challenges & Key Takeaways (My Struggles)
Building a fully decoupled system forced me to develop critical debugging and problem-solving skills to fix these problems:

**1. Data Cleaning Problem**: When I tested my models, I realized there was an overfitting problem to the model, and I tried to analyze the problem by rechecking the dataset, where I found there are multiple duplicates of rows. In th data, some rows had the same data even though  the column "Person ID" was different.
Another challenge is **Feature Engineering**, where I dropped useless features and remap some features, one problem is a BMI feature which only includes 3 unique values which are "Normal Weight","Normal", "Overweight", "Obese", where I merged and map "Normal Weight" and "Normal" into a same value, and mapped the rest to a different value.

**2. Generating Backend API and Frontend codes with AI help**: I used AI help to generate the main API codes and Frontend codes, but AI had some problems such as wrong variable inputs and returns, so I had to manually recheck the codes, find and fix some errors.

##  Machine Learning Core & Performance
The predictive system is powered by an **XGBoost Regressor** trained in a multi-output configuration to predict continuous target values simultaneously.
- **R² Score:** `0.90` (The model successfully accounts for 90% of the variance in the training data)
- **Mean Squared Error (MSE):** `0.18` (Indicates a very tight fit with minimal prediction errors)

**Some Weaknesses** : Even though the model has a high score of predicting values, but the data sources are limited. For example : Most of the data are for Adults who has occupations and age ranges from their 20s to their 60s. Data may not be accurate for your situation.
